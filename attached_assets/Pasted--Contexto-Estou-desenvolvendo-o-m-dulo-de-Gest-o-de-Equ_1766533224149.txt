# Contexto
Estou desenvolvendo o módulo de "Gestão de Equipe" do meu SaaS Financeiro (React + ShadcnUI + Node/Express).
O usuário logado é um "Admin da Empresa" e está gerenciando os funcionários da sua própria empresa.

# Objetivo
Criar a página `src/pages/settings/Team.jsx`.
Esta página deve listar os usuários da empresa, permitir adicionar novos e editar permissões granulares.

# Requisitos Visuais e Funcionais

## 1. Listagem de Usuários (Tabela)
- Colunas: Avatar + Nome, Email, Cargo (Badge), Status (Ativo/Pendente) e Ações (Editar/Excluir).
- Se o usuário for o próprio Admin logado, bloquear botão de excluir (não pode se auto-deletar).

## 2. Modal de "Adicionar Membro" (Dialog)
Este modal deve ter DUAS abas (Tabs do shadcn/ui):

### Aba A: "Cadastro Direto"
Cria o usuário imediatamente.
- Campos: Nome, Email, Senha Provisória.
- Switch: "Usuário é Admin?" (Se sim, dá acesso total. Se não, mostra as permissões abaixo).

### Aba B: "Enviar Convite" (Link Mágico)
Gera um link para o funcionário se cadastrar sozinho.
- Campos: Email do funcionário.
- Ação: Botão "Gerar Link".
- Resultado: Mostra um input readonly com o link (ex: `app.com/invite/token-123`) e um botão "Copiar Link" e outro "Enviar por Email".

## 3. Seção de Permissões Granulares (Checkboxes)
Esta seção aparece no Modal de Criação (se for Operacional) e no Modal de Edição.
Use Switches ou Checkboxes para definir o que este usuário pode fazer. Salve isso num objeto JSON.

Lista de Permissões para incluir:
- [ ] Ver Dashboard Financeiro (Saldos)
- [ ] Gerenciar Transações (Criar/Editar receitas e despesas)
- [ ] Gerenciar Clientes e Fornecedores
- [ ] Acessar Relatórios Avançados (DRE, Lucro)
- [ ] Importar Extrato Bancário
- [ ] Calculadora de Preços

# Lógica de Integração (Mock)
- Simule as chamadas de API com `fetch`.
- GET `/api/team`: Lista usuários da empresa atual.
- POST `/api/team`: Cria usuário direto.
- POST `/api/team/invite`: Gera o link de convite.
- PATCH `/api/team/:id`: Atualiza permissões.
- DELETE `/api/team/:id`: Remove usuário.

# Detalhes de UX
- Use `sonner` para toasts de sucesso ("Link copiado!", "Usuário criado!").
- Use ícones `lucide-react`.
- O layout deve ser responsivo.
