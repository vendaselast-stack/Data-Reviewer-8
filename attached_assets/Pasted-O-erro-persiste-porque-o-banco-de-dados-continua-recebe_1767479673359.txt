O erro persiste porque o banco de dados continua recebendo dados em formatos que ele não consegue processar, especificamente relacionados a valores numéricos e identificadores (IDs).

Ao analisar as imagens e os logs das queries, os problemas centrais são:

1. Formato de Moeda Inválido (Vírgula vs. Ponto)
O PostgreSQL e a maioria dos bancos de dados SQL esperam que campos do tipo numeric ou decimal usem o ponto (.) como separador decimal e não contenham separadores de milhar ou símbolos de moeda.

O Erro: Nas imagens, o valor enviado é 515454 (ou interpretado como texto com vírgula internamente).

A Causa: Se o seu front-end envia o valor formatado como 5.154,54, o banco dispara o erro invalid input syntax for type numeric porque a vírgula e o ponto de milhar não são caracteres numéricos válidos para esse campo.

2. Conflitos de Chaves Estrangeiras (IDs)
Na segunda imagem, a query tenta inserir na tabela transactions.

Parâmetros Incompletos: O log mostra um UUID (9dba4541...) seguido de campos como compra, 545454, e fsdfdff.

Problema de Relacionamento: Se o customer_id ou supplier_id enviado não existir exatamente como está escrito na tabela de origem, a query falha por violação de integridade referencial.

3. Diferença entre as Tabelas purchases e transactions
Você está tentando registrar uma "Compra", mas o sistema parece estar tentando inserir dados em duas tabelas diferentes em momentos distintos:

Imagem 1: Tenta inserir em purchases.

Imagem 2: Tenta inserir em transactions. O erro ocorre porque a estrutura de colunas ou os tipos de dados enviados para transactions (como o campo amount) não estão saneados.

Como corrigir agora:
Sanitize os inputs numéricos: No código do seu "Botão Registrar", você deve garantir que o valor seja convertido para um número puro antes da query.

Errado: "5.154,54" (String)

Certo: 5154.54 (Number/Float)

Verifique campos obrigatórios: Certifique-se de que nenhum campo numérico está sendo enviado como string vazia (""), o que também causa esse erro específico no Postgres; envie 0 ou NULL (sem aspas) se o campo permitir.

Ajuste a ordem dos parâmetros: No log da imagem 2, parece que o valor 545454 está sendo passado em uma posição de coluna que talvez não seja a de amount. Verifique se a ordem dos params no seu código bate exatamente com a ordem das colunas no INSERT INTO.
