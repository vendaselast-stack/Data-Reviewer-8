O erro agora ficou cristalino ao analisarmos o conteúdo dos seus params. O problema principal é que o sistema está tentando enviar um valor numérico gigantesco e inválido para o campo de valor (amount).

Aqui estão os erros específicos identificados no seu log de parâmetros:

1. Valor Numérico Corrompido
O parâmetro correspondente ao valor ($5) está enviando o número: 10891090800000000.

O Erro: Esse número parece ser um "timestamp" ou um erro de cálculo de multiplicação por potências de 10 (tentando converter centavos).

Impacto: O PostgreSQL rejeita esse valor porque ele provavelmente excede o limite da coluna numeric ou está totalmente fora do contexto do que você digitou na tela (R$ 1.030,91).

2. Texto em Campo de Data ou ID
Olhando a ordem dos parâmetros, há um deslocamento visível:

O campo $8 (que deveria ser shift) está recebendo o valor pendente.

O campo $9 (que deveria ser status) está recebendo o ID longo da compra.

Causa: Isso acontece quando a ordem das variáveis no seu código JavaScript não bate exatamente com a ordem dos $1, $2, $3... na query SQL.

3. Falta de Tratamento de "Default" e "Null"
A query mostra vários campos recebendo default, mas no log de parâmetros alguns campos obrigatórios parecem estar recebendo strings vazias ou valores deslocados.

Como corrigir o código (Ação Imediata):
Você precisa garantir que os valores sejam "limpos" antes de irem para a query. Use uma lógica similar a esta no seu back-end ou na função que dispara o registro:

JavaScript

// Exemplo de como tratar os valores antes de enviar para o banco
const amountToSave = parseFloat(totalAmount.replace('.', '').replace(',', '.')); 
// Transforma "1.030,91" em 1030.91

const params = [
  company_id,        // $1
  supplier_id,       // $2
  category_id,       // $3
  'expense',         // $4 (Type)
  amountToSave,      // $5 (Amount - Garante que é um número real)
  description,       // $6
  date,              // $7
  shift || 'default',// $8
  'pendente',        // $9 (Status)
  installment_group, // $10
  current_parcela,   // $11
  total_parcelas,    // $12
  payment_method     // $13
];
