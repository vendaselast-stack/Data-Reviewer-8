# Contexto do Projeto
Estou desenvolvendo um SaaS Financeiro Multi-tenant (Multi-inquilino).
Stack Tecnológica: React, Tailwind CSS, Shadcn/UI, React Hook Form, Zod (validação), Lucide React (ícones) e Tanstack Query.

# Objetivo
Preciso que você crie o componente da página de "Perfil do Usuário" (`src/pages/settings/Profile.jsx`). Esta página é onde o usuário logado visualiza e edita seus dados pessoais.

# Requisitos Visuais (UI/UX)
1. Layout:
   - Use um layout de "Configurações" moderno.
   - Esquerda: Um menu lateral simples de navegação (links: "Meu Perfil", "Segurança", "Notificações") - O foco agora é a aba "Meu Perfil".
   - Direita: O conteúdo do formulário dentro de um Card principal (shadcn/ui Card).

2. Componentes (Shadcn/UI):
   - Use `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`.
   - Use `Input` para campos de texto.
   - Use `Button` para ações.
   - Use `Avatar` para a foto de perfil.
   - Use `Separator` para dividir seções.
   - Use `Sonner` ou `Toast` para feedback de sucesso/erro.

3. Seção de Avatar (Destaque):
   - Deve mostrar o Avatar atual em tamanho grande (h-24 w-24).
   - Ao lado do avatar, um botão "Alterar Foto" e um texto de ajuda ("JPG ou PNG, max 2MB").
   - Funcionalidade de Preview: Quando o usuário selecionar um arquivo, a imagem do avatar deve atualizar instantaneamente na tela (preview local) antes mesmo de salvar.

# Estrutura de Dados e Campos

## Seção 1: Identificação (Read-Only por Segurança)
Estes campos vêm do banco e NÃO podem ser editados pelo usuário para garantir a integridade do SaaS. Devem aparecer visualmente como desabilitados (`disabled` e fundo cinza claro).
- E-mail (ex: usuario@empresa.com)
- Cargo/Role (ex: Admin ou Operacional)
- Empresa (ex: Nome da Empresa Vinculada)

## Seção 2: Dados Pessoais (Editáveis)
- Nome de Exibição (Display Name)
- Telefone / WhatsApp (Com máscara se possível)

## Seção 3: Endereço Completo (Editáveis)
Agrupe estes campos visualmente:
- CEP (Zip Code)
- Rua (Street)
- Número
- Complemento
- Bairro
- Cidade
- Estado (UF - max 2 chars)

# Requisitos Funcionais e Lógica
1. State Management:
   - Use `react-hook-form` para gerenciar o estado do formulário.
   - Use `zod` para validar os campos (ex: nome obrigatório, telefone com mínimo de caracteres).

2. Carregamento de Dados (Fetch):
   - Ao montar a página (`useEffect`), simule uma chamada à API `/api/auth/me` ou `/api/profile` para preencher os campos.
   - Mostre um estado de "Skeleton" ou "Loading" enquanto os dados carregam.

3. Salvamento (Submit):
   - O envio deve suportar upload de arquivo.
   - Se houver imagem nova, o body da requisição deve ser `FormData` (multipart/form-data).
   - Se não houver imagem, envie JSON padrão ou trate tudo como FormData.
   - Exiba uma notificação de "Sucesso" ou "Erro" após a promessa resolver.

# Regras de Negócio Importantes
- O usuário NUNCA pode alterar seu próprio `role` (cargo) ou `company_id` através deste formulário.
- O botão "Salvar" deve mostrar um estado de "loading" (spinner) durante a requisição para evitar duplo clique.
