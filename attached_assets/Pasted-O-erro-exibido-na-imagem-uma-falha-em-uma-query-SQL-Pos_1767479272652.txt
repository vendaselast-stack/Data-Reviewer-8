O erro exibido na imagem é uma falha em uma query SQL (PostgreSQL) ao tentar registrar uma nova compra. Embora o modal mostre a query e os parâmetros, o motivo exato da falha geralmente está escondido no "back-end", mas com base nos dados visíveis, aqui estão os motivos mais prováveis:

1. Conflito de Formato de Dados (Data Type Mismatch)
O erro invalid input syntax for type numeric é muito comum quando se tenta enviar um texto para um campo numérico.

Preço com vírgula: No formulário, o valor está como 5.154,54 (padrão brasileiro). No entanto, o banco de dados espera o padrão americano 5154.54 (com ponto). Se o seu sistema não estiver tratando essa conversão antes de enviar ao banco, a query falhará.

Campos Vazios: Se algum campo obrigatório (como paid_amount) estiver sendo enviado como uma string vazia ("") em vez de 0 ou NULL, o PostgreSQL rejeitará a inserção.

2. Erro de Identificador de Relacionamento (IDs Inválidos)
Nos parâmetros exibidos (params), vemos vários UUIDs (como 9dba4541...).

Chave Estrangeira (Foreign Key): O erro pode estar ocorrendo porque o supplier_id (ID do fornecedor) ou o category_id não existe na tabela de referência. Por exemplo, se você selecionou a categoria "Compras", mas o ID enviado não consta na tabela de categorias, o banco bloqueia a ação para manter a integridade.

3. Violação de Restrição (Constraint Violation)
Valores Nulos: A query tenta inserir em colunas como company_id, supplier_id e description. Se alguma dessas colunas estiver configurada como NOT NULL no banco e o valor enviado for nulo ou inválido, a query falha.

Como Resolver:
Verifique os Logs do Servidor: O modal apenas diz que a query falhou. O log do seu back-end (Node.js, Python, etc.) terá a mensagem de erro específica do Postgres (ex: ERROR: column "amount" is of type numeric but expression is of type text).

Trate o valor monetário: Certifique-se de que a máscara de dinheiro no front-end seja convertida para um número puro antes de ser enviada (ex: 5.154,54 vira 5154.54).

Cheque os IDs: Confirme se o fornecedor e a categoria selecionados realmente possuem IDs válidos vinculados a eles no banco de dados.
